User Story: 
As an admin, I want to get a quick assessment of whether audio material has been edited before the upload.	

Acceptance Criteria (generated from Model #1)
1. The system should provide an automatic check to verify if the audio material has been edited before the upload.
2. If the audio material has been edited, the system should display a notification to the admin indicating that it has been edited.
3. If the audio material has not been edited, the system should not display any notification to the admin.
4. The system should perform the check for edited audio material in real-time or within a reasonable time frame after the upload.
5. The check for edited audio material should be accurate and reliable, minimizing false positives and false negatives.
6. The admin should have the ability to manually override the result of the check if necessary.

Acceptance Criteria (generated from Model #2)
1. The system should have an audio upload functionality for admins.
2. Upon uploading an audio file, the system should analyze the audio to determine if it has been edited before.
3. The analysis should check for various edits such as cuts, silence removal, and noise reduction.
4. If the audio has been edited, the system should provide a visual indication or flag indicating that it has been edited.
5. The analysis should be performed in real-time or within a reasonable time frame after the upload.
6. The system should provide an option for admins to download a report that details the analysis results for each uploaded audio file.
7. The analysis results should be accurate and reliable, minimizing false positives or false negatives.
8. The system should support various audio file formats commonly used in the industry.
9. The analysis should take into account different types of edits commonly found in audio material, such as splicing, pitch correction, and time stretching.
