User Story: 
As a depositor, I want to deposit arbitrarily large files, so that I am not limited in what files I can and cannot deposit.

Acceptance Criteria (generated from Model #1)
1. The system should allow the depositor to upload files of any size, without imposing any arbitrary limits.
2. The uploading process should provide a progress indicator to show the progress of the file upload.
3. After the file is uploaded, the system should provide a confirmation message to the depositor.
4. The system should properly handle and notify the depositor in case of any errors or failures during the file upload process.
5. The depositor should be able to view the uploaded file after it has been successfully deposited.
6. The system should ensure the security and integrity of the deposited files, protecting them from unauthorized access or modifications.
7. The depositor should be able to deposit various types of files, including but not limited to documents, images, videos, and audio files.
