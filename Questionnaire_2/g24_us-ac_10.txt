User Story: 
As a depositor, I want to have a user interface that is familiar to me, so that I feel like all the University systems are joined up.

Acceptance Criteria (generated from Model #1)
1. The user interface should have a consistent layout and navigation across all university systems.
2. The user interface should utilize common design patterns and conventions that are familiar to the target audience.
3. The user interface should have a similar look and feel to the university's branding and visual identity.
4. The user interface should provide clear and intuitive labels, icons, and controls to facilitate easy navigation and interaction.
5. The user interface should maintain a cohesive user experience by following established usability principles.
6. The user interface should be responsive and adaptive, ensuring compatibility across different devices and screen sizes.
7. The user interface should limit the learning curve for users familiar with other university systems, enabling them to quickly navigate and complete tasks.
8. The user interface should provide seamless integration and data flow between different university systems, reducing the need for duplicate data entry.
